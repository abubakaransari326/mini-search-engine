Boundary Testing

Important boundary cases include empty query, unknown term, and query with only stop words. You should also test whitespace only input, punctuation only input, repeated query tokens, and multi keyword queries whose terms never co occur in the same document.

Additional tests should verify repeated terms, punctuation cleanup, and case normalization. Normalization should map uppercase letters to lowercase and strip punctuation so that Crawler and crawler converge to the same token in the index.

Testing validates correctness and makes demo output more convincing. Capture sample runs in a text file so reviewers can reproduce your transcript without guessing which queries you typed during the video recording.

Finally, test hyperlink extraction: ensure anchor tags with relative paths are discovered, while mailto links and fragment only anchors are ignored or handled according to your project specification. Consistent rules reduce surprises during the crawl phase.
