Mini Search Portal

This small website contains pages about algorithms, data structures, web crawling, ranking, software testing, and practical parsing tools. The collection is intentionally compact so you can trace every step from crawling to indexing to query answering.

The search engine project should index all meaningful terms while excluding common stop words. Students can experiment with single keyword queries, multi keyword conjunction queries, and simple relevance ranking based on term frequency within each document.

Hyperlinks connect these pages into a tiny graph. A crawler can begin at this home page, enqueue neighbors, dequeue the next uniform resource locator, and avoid cycles by tracking visited document identifiers. The resulting crawl order is easy to reason about on paper.

When demonstrating the system, show both typical queries and boundary conditions such as unknown vocabulary, punctuation only input, whitespace only input, and queries that contain solely stop words after normalization.
